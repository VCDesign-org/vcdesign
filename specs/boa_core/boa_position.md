# Positioning of BOA

## Role
BOA (Boundary-Oriented Architecture) is **one possible valid implementation** of the VCDesign Core.

## Mechanism vs. World
- **BOA is Mechanism.** It defines boundaries, agents (RCA), and protocols (RP).
- **BOA is NOT World.** It does not define "good" or "bad". It only defines "Signed" or "Unsigned".

## Independence
BOA functions as a "Cold Mechanism". It does not care *why* a judgment is signed, only *that* it is signed by a responsible actor. This allows BOA to remain valid even as social ethics (Worldview) shift.
