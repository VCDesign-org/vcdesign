# Governance of UVW

UVW is the "Problem Space definition". Changing the problem space is a fundamental shift.

## Breaking Changes
Adding or removing an **Axiom** is a Breaking Change.
- If the world condition changes, all Responses (VCDesign, etc.) are invalidated.
- Therefore, UVW is extremely stable and should rarely change.

## Non-Breaking Changes
- Clarifications in Glossary.
- Adding examples to Compatibility.
- Refining the `non_assumptions.md` list.
