VMS Marketing

Marketing Judgment Support Extension

VMS Marketing は、VCDesign の意思決定原則を保ったまま、 マーケティング相談を構造化・比較・留保・説明可能性の形に変換する知識拡張レイヤです。

Position

VMS Marketing は VCDesign 本体ではありません。 VMS の制約に従いながら、マーケティング領域に特化した相談支援を定義するサブセットです。

目的は、AI にマーケティング判断を自動化させることではありません。 人間の判断を支えるための問い、構造、比較軸、不確実性、説明可能性を与えることです。

Interpretation

相談が何を求めているか、どの段階の問いかを正規化する。

Proposal

複数の候補案とトレードオフを残し、比較可能な判断材料にする。

Traceability

何を見て、どの構造で考え、どこが未確定だったかを記録する。

Marketing Domains

相談は単一の正解へ急がず、1〜4 個の主要ドメインへ重み付きで振り分けます。 ドメインは回答そのものではなく、参照する観点と問いの棚です。

Boundary

どの市場、顧客、用途、価格帯で戦うのか。

Meaning

ブランドは何者として認識されるべきか。

Interaction

顧客体験のどこで理解・離脱・継続が起きているか。

Transmission

どう伝えると理解され、広がり、誤解されにくいか。

Execution

売り方、販路、継続率、運用負荷をどう設計するか。

Validation

仮説の思い込み、不確実性、検証すべき現実反応は何か。

Processing Flow

1. 問いの正規化

何を求めているか、どの段階の相談か、期待する出力形式は何かを把握する。

2. ルーティング

router.yaml に従い、関連ドメインを重み付きで選ぶ。

3. Judgment Proposal

facts、assumptions、candidate proposals、unknowns、routing trace を生成する。

4. 知識参照

理論を要約するのではなく、問い・比較軸・観点へ変換して使う。

5. 回答生成

構造、選択肢、トレードオフ、推奨方向、不明点、次の問いを出す。

6. ログ化

answer_log.yaml と整合する形で、理由と不確実性を残す。

この拡張で追加される扱い

  • Expiry: 戦略・市場・ブランド・販路などの中期判断に有効期限を持たせる
  • Reality Drift: 実行後の現実反応を、意味仮説との乖離として扱う
  • Resolution Handshake: 採用判断の材料を整え、最終決定は人間に残す

回答テンプレート

要約:
  - この相談は何についての問いか
構造の見立て:
  - どのドメインの問題か
  - 主要な論点は何か
選択肢:
  - Option A
  - Option B
  - 必要なら Option C
推奨方向:
  - 現時点ではどれを優先するか
  - なぜそう考えるか
不明点:
  - まだ確認できていない点
次の問い:
  - 次に何を確認すれば判断が進むか

Guardrails

良い回答とは、派手なアイデアではなく、問題の構造が見え、 選択肢の差が分かり、unknown が可視化され、次の判断がしやすい回答です。

  • AI は semantic loop に位置し、最終判断を下さない
  • 事実と推定を分離する
  • 不明点を無理に埋めない
  • 比較案なしに 1 案へロックインしない
  • 回答理由と参照構造を追跡可能にする

Resources

VMS Marketing は、マーケティング相談支援 AI に読み込ませる仕様群です。 本体 VMS と併用し、マーケティング領域の知識拡張として扱います。

生成 AI 向けの最小読込セット

  • specs/engine_config.yaml 全体構成と処理パイプライン
  • specs/router.yaml 質問をマーケ知識ドメインへ振り分けるルータ
  • specs/marketing_knowledge_protocol.yaml ドメイン別の理論・観点・問い
  • specs/judgment_proposal.yaml 判断可能な単位への分解
  • specs/answer_log.yaml 回答根拠、不明点、信頼度のログ形式
  • system_prompt.md マーケティング相談支援 AI の基本指示
  • VMS Marketing GitHub Repository VMS Marketing の仕様リポジトリ。YAML と system prompt の正本はこちらを参照してください。
