Reality Fit

Adaptive Loop Model

VCDesign の Core / Policies / Metrics が、現実のAI運用ループへどう接続するかを説明します。

位置づけ

このページはVCDesignの現行authorityを説明する補助ページです。 authorityは core/core.yaml、core/policies.yaml、core/metrics.yaml のままであり、 このページは第四のauthorityではありません。

4つのループ

  • Physical Loop: 実行、安定性、連続系ダイナミクス
  • Semantic Loop: 解釈、delta-to-meaning変換、非連続推論
  • Value Loop: 目的、制約、最適化方向
  • Responsibility Loop: 最終判断、停止、説明責任

VCDesignは、適応をこれらのループ間の責任付き協調として扱います。

これは PDΔA と並ぶ独立制御モデルではありません。PDΔA を時間スケールと責任配置に沿って分解した説明です。

時間スケール

各ループは異なる時間定数で動きます。高速な実行ループが、低速な価値判断や責任判断を書き換えてはなりません。

LLM の配置

LLMは主としてSemantic Loopに配置されます。解釈や推奨は許可されますが、 Physical Loopの直接制御、価値目的の自律設定、Responsibility Loopの最終権限保持は許可されません。

境界と停止可能性

非連続推論と連続制御の間にある越境には、安全ゲート、追跡可能性、人間override、停止経路が必要です。

Boundary は単なる変換層ではなく、責任を明示的に再確認または再配置しなければならない点でもあります。

成長の定義

成長は単なるモデル性能の向上ではありません。 責任制約下での運用整合の改善として定義されます。

Authority との対応

  • Core: ループ分離、時間スケール分離、境界媒介
  • Policies: LLM配置制約、安全ゲート、停止可能性
  • Metrics: delta品質、整合性品質、一貫性、停止準備度
